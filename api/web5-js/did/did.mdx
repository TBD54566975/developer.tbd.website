# did
## Overview
The `did` class is responsible for providing all operations related to DIDs.

## Methods

### create(method, options)

The `create` method under the `did` scope enables generation of DIDs for a supported set of DID Methods. The output is method-specific, and handles things like key generation and assembly of DID Documents that can be published to decentralized DID networks.

#### Parameters

`method`

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Specifies the method on which to create the did.

`options` (Optional)

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Enables customization of the did formation.

#### Return Value

Returns a did document.

#### Code Example

```js
const myDid = await web5.did.create('ion');
```

### encrypt(options)

Encrypts provided data according to specification

#### Parameters
`options`

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; `did` - DID of the recepient, whose private key is used to decrypt the data.

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;`payload` - Encryption algorithm output, including the ciphertext, to be decrypted.

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;`keyId` - Key identifier to use form the recepient's DID documen.

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;`algorithm` - An object specifying the algorithm to be used and any extra parameters if required.

#### Return Value
`Promise<Web5EncryptionResult>` 

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;A `Promise`that fulfills with the output of the encryption algorithm.

#### Code Example
```js
const options = {
    "did" : "firstDid",
    "payload" : "decodePayload"
    "keyId" : "sampleKeyId",
    "algorithm' : "sampleAlgorithm"
}
const encryptedText = await web5.did.encrypt(options)
```
