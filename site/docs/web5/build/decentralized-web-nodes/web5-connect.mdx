# Web5.connect()

The Web5 JS SDK provides a simple interface to get started building Web5 applications quickly. It provides APIs to use [Decentralized Identifiers (DIDs)](/docs/web5/learn/decentralized-identifiers) and [Decentralized Web Nodes (DWNs)](/docs/web5/learn/decentralized-web-nodes) right away by initializing the SDK through `Web5.connect()`.
<br />

:::caution
WEB5 JS SDK IS CURRENTLY IN TECH PREVIEW

The SDK is currently still in Tech Preview and under active development. Additional functionality will be added in the lead up to version 1.0, and modifications will be made to address issues and community feedback.
:::

## Connect

Using `Web5.connect()`, an instance of Web5 is created that enables an app to connect to an existing decentralized identifier (DID) either by direct creation or connection to an [identity agent app](/docs/web5/learn/agents) or create a new one. The method also creates a locally running Decentralized Web Node (DWN) and associates it with the DID.

This local DWN is synchronized with a remote DWN node (if specified) that enables connectivity and sync across the user's devices and other users. While the Web5 JS SDK is in technical preview, a remote DWN is automatically provisioned and connected to the local DWN - syncing every 2 minutes.

<CodeSnippet functionName="createDidAutomatically" />

### Options

Enables an app to request connection to a user's local identity app (like a desktop or mobile [agent](/docs/web5/learn/agents) - work is underway for reference apps of each), or generate an in-app DID to represent the user (e.g. if the user does not have an identity app)

:::note
The outputs of this method invocation will be used throughout the other API methods.
:::

## Upgrade your application to a PWA
Upgrade your app to a PWA and use Web5 activated polyfills to add Decentralized Resource Locators(DRLs) and manage DWN offline syncs. See [DWA starter app](https://github.com/TBD54566975/dwa-starter)
```js
// Activate Web5 Service Worker Polyfills 
activatePolyfills({ 
  onCacheCheck() { return { ttl: 30000, }; 
  },
});
```

## Examples
Below are examples of how to use `Web5.connect()` with or without additional option parameters.

##### Connect with no parameters
<CodeSnippet functionName="createDidAutomatically" />

##### Connect to DIF/Google Community Node

Connect to the community node instance hosted by Google and DIF:

```js
const {did, web5} = await Web5.connect({
  didCreateOptions: {
    dwnEndpoints: ['https://dwn.gcda.xyz'], // User provides google's community DWN instance
  },
  registration: {
    onSuccess: () => {
      // Registration succeeded, set a local storage value to indicate the user is registered and registration does not need to occur again.
    },
    onFailure: (error) => {
      // Registration failed, display an error message to the user, and pass in the registration object again to retry next time the user connects.
    },
  },
})
```
