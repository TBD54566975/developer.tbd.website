# Web5 CLI

The Web5 CLI is a Rust-based command-line tool for creating and managing [Decentralized Identifiers (DIDs)](/docs/web5/decentralized-identifiers/what-are-dids) and [Verifiable Credentials (VCs)](/docs/web5/verifiable-credentials/what-are-vcs).

This guide will walk you through installation and usage of the Web5 CLI.

## Installation

To install the Web5 CLI, run:

```bash
curl -L https://raw.githubusercontent.com/TBD54566975/web5-rs/main/crates/web5_cli/install.sh | bash -s v0.0.4
```

## Creating a DID

The `web5 did create` command creates a DID using one of the supported DID methods listed below. 

```bash
web5 did create <method> [options]
```

**Supported DID methods**
- `did:dht`: [Distributed Hash Table](/docs/glossary#did-method---dht)
- `did:jwk`: [JSON Web Key](/docs/glossary#did-method---jwk)
- `did:web`: [Web](/docs/glossary#did-method---web)

**Options**
* `--no-indent`: Output the DID document without indentation.
* `--json-escape`: Escape JSON strings in the output.
* `--domain <domain>`: For did:web, specify the domain for the DID Web creation.
* `--no-publish`: For the dht method, prevents publishing the DID immediately.

**Examples**

```bash
# Create a did:jwk with no indent and json escape
web5 did create jwk --no-indent --json-escape

# Create a did:web 
web5 did create web --domain https://example.com

# Create a did:dht without publishing
web5 did create dht --no-publish
```

## Resolving a DID

The `web5 did resolve` command [resolves a DID](/docs/web5/decentralized-identifiers/how-to-resolve-a-did).

```bash
web5 did resolve <did-uri> [options]
```

**Options**
* `--no-indent`: Output the DID document without indentation.
* `--json-escape`: Escape JSON strings in the output.

**Examples**

```bash
web5 did resolve did:dht:bwrh9fphhegh1jn3hhe656zuyfxqdm493so5r4hmpte1yf456w5o
```

## Create and Sign a VC
The `web5 vc create` command [creates and sign VCs](/docs/web5/verifiable-credentials/vc-issuance).

```bash
web5 vc create <credential-subject-id> [options]
```

**Options**
* `--portable-did <did>`: specifies the DID used to sign the Verifiable Credential. If omitted, VC will be unsigned
* `--issuer <issuer-did>`: Specifies the issuer of the credential. If issuer is provided issuer without `--portable-did`, the VC will be unsigned.
* `--expiration-date <date>`: Set an expiration date for the credential.
* `--no-indent`: Output the VC without indentation.
* `--json-escape`: Escape JSON strings in the output.

**Examples**
```bash
## Portable DID that will be used as the issuer and to sign the credential
export PORTABLE_DID=$(web5 did create dht --no-indent)

## Create and sign VC with portable DID
web5 vc create "did:dht:cm1yqfjzfdtauh33nauwf3sqsijqziwrydicr8dbtho3cucb9nno" --portable-did $PORTABLE_DID

## Create and sign VC with an expiration date
web5 vc create "did:dht:cm1yqfjzfdtauh33nauwf3sqsijqziwrydicr8dbtho3cucb9nno" --portable-did $PORTABLE_DID --expiration-date 2025-12-31
```
